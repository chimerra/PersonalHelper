# Тестовые данные

Набор для варианта **«от текста до действия»** (capture → задачи/заметки).

## Файлы

| Файл | Назначение |
|------|------------|
| `inputs.jsonl` | 11 входных текстов (как ручной ввод в UI) |
| `specs.jsonl` | Ожидаемое поведение для каждого кейса |

Другие форматы (`queries.jsonl`, `kb_documents.jsonl` и т.п.) относятся к другим вариантам проекта; для capture достаточно **inputs + specs**.

## Требования (чеклист)

- [x] Папка `tests_data/` с тестами под выбранный вариант
- [x] Минимум **10** тестов в `inputs.jsonl`
- [x] **2** кейса с ожиданием `needs_review=true` (№1 и №4 в `specs.jsonl`)

## Запуск

```bash
python -m app.seed
```

Каждая строка `inputs.jsonl` прогоняется через **`capture_text`** (тот же pipeline, что кнопка «Создать элемент»). После прогона seed сверяет результат с `specs.jsonl`.

Для чистого прогона: «Очистить базу» в UI или удалите `data/app.db`.  
Повторный запуск пропускает уже сохранённые тексты (дедупликация).

В Docker:

```bash
docker compose exec app python -m app.seed
```

## Кейсы (11)

| id | Суть | needs_review | Что проверяем |
|----|------|:------------:|---------------|
| 1 | «надо что-то сделать с жизнью» | **да** | слишком общий ввод |
| 2 | книга «Атомные привычки» | нет | чтение / заметка |
| 3 | пикник или фильм дома | нет | условие → 2 задачи |
| 4 | зал / дома / отдых? | **да** | неопределённость, выбор |
| 5 | список рабочих дел | нет | multi-task + ignored |
| 6 | курсовая + проект, завтра | нет | несколько задач, срок |
| 7 | YouTube про помидор | нет | URL в заметке |
| 8 | идея геймификации | нет | заметка-идея |
| 9 | презентация к встрече завтра | нет | срочность, дедлайн |
| 10 | природа + подарок + код | нет | 3 задачи из одного текста |
| 11 | поток про тег «работа» = важное | нет | **автопамять**, structured key |

### Как попасть в память через capture (№11)

Текст — **поток сознания** с устойчивым правилом, не разовая задача. Маркеры: «обычно», «я предпочитаю», «с тегом …», «важные/high».

Пример из №11:
```text
слушай, пока ковыряюсь с этим помощником... у меня так завелось: всё по работе — с тегом работа,
и я обычно такие штуки считаю важными. это не про одну задачу, просто привычка, запомни.
```

Ожидаемый факт: `priority_for_tag_работа` → `тег работа → high priority` (не сырой `user_preference`).

Точные заголовки и типы могут слегка отличаться при ответе ИИ; seed не подставляет готовые карточки. Критичные сценарии (особенно **needs_review**) стабильны и на fallback без API-ключа.

## Формат inputs.jsonl

```json
{"id": 1, "text": "…", "user_id": "u_1"}
```

## Формат specs.jsonl

```json
{"id": 1, "title": "…", "expect_needs_review": true, "expect_min_items": 1, "scenario": "vague_life_goal"}
```
